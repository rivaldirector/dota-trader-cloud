# Отчёт: Dota 2 Quant Trading Model — Анализ предварительного edge

## Контекст проекта

Мы строим paper-trading систему для ставок на Dota 2 матчи. Цель — доказать или опровергнуть наличие положительного edge против букмекерского рынка.

**Стек:**
- Данные матчей: PandaScore API (~12 месяцев исторических матчей Dota 2)
- Данные коэффициентов: BetsAPI odds/summary (~5 недель opening/closing odds)
- Предиктивная модель: Walk-forward Elo (K=32, tier-weighted K, time decay 365-day half-life)
- БД: SQLite (таблицы matches, odds_snapshots, bets)

---

## Параметры модели (текущая версия)

- **Алгоритм:** Elo rating, walk-forward (без look-ahead bias)
- **Начальный рейтинг:** 1500 для всех команд
- **K-фактор:** 32 базовый, умножается по tier:
  - TI/Major (Tier S) → K × 1.5
  - DPC/ESL (Tier A) → K × 1.2
  - Regional qualifier (Tier C) → K × 0.7
- **Time decay:** `weight = 0.5^(days_ago / 365)` — матч год назад весит 0.5
- **Предсказание:** Логистическая функция от разницы Elo

**No-vig вероятность рынка:**
```
p_home = (1/odd_home) / (1/odd_home + 1/odd_away)
```

**Edge:**
```
edge_open  = model_prob - market_novig_prob_at_open
edge_close = model_prob - market_novig_prob_at_close
```

**CLV (Closing Line Value):**
```
CLV = количество матчей где edge_close < edge_open
    = рынок движется в сторону нашей модели после открытия
```

## Ключевые метрики

### Edge статистика

| Метрика | Значение | Интерпретация |
|---|---|---|
| Средний edge_open | **-0.019** | Модель в среднем на 1.9% хуже рынка |
| Средний edge_close | **-0.005** | Рынок немного движется к модели к закрытию |
| Матчей с +edge_open | 98 из 228 (43%) | |
| WR при +edge_open | **48.0%** | Ниже 50% — тревожный сигнал |
| WR при -edge_open | **63.8%** | Когда модель думает рынок прав — рынок прав |
| CLV | **14%** (32/228) | Критически низкий |

### CLV интерпретация

CLV > 50% означает что модель "опережает рынок" — рынок после открытия движется туда куда мы предсказали. Это главный индикатор качества модели.

Наш CLV = 14% означает: рынок в 86% случаев движется ПРОТИВ нашей ставочной позиции. Модель систематически ошибается о направлении движения линии.

## Гипотезы о причинах слабого edge

### 1. Калибровка — главная проблема

Elo даёт вероятности в диапазоне примерно 0.35–0.75. Рынок ставит 0.80–0.96 на явных фаворитов. 

Пример: если Elo говорит 0.70 на Team A, а рынок ставит 0.90 — мы видим "+edge = 0.20". Но если Team A действительно должна выиграть с вероятностью 0.90 (и побеждает в 90% случаев), то наш "edge" иллюзорный.

Elo-разница, дающая 0.70 вероятности по формуле, в реальности может соответствовать 0.90 для Dota 2 (из-за специфики игры, нелинейности разрыва между топами и аутсайдерами).

### 2. Информация которой нет в Elo

- Состав команды (ростер): замена 1-2 игроков меняет силу на 20-30%
- Патч Dota: сильные команды по одному патчу могут упасть после нового
- Форма на турнире: momentum эффект
- Усталость: плотный турнирный календарь

### 3. Аномальные closing odds (фильтрация нужна)

Видны строки где |close - open| > 0.30:
- open=0.500, close=0.955 — скорее всего снятие ставок на одну сторону (форс-мажор)
- Эти матчи искажают статистику CLV

### B. Фильтрация аномальных матчей

Убрать из расчёта edge матчи где:
- |close_prob - open_prob| > 0.30
- Один из коэффициентов < 1.05 (монополия — мало букмекеров)

### C. Расширение фич

Добавить к Elo:
- last5_winrate (уже считается в team_rating.py, не используется в предсказании)
- days_since_last_game (свежесть команды)
- head_to_head history (H2H при наличии данных)

### D. Модель второго уровня

Поверх Elo обучить логистическую регрессию или XGBoost с фичами:
- elo_diff (разница Elo)
- elo_prob (предсказание Elo)

- market_open_prob (рыночный сигнал как фича)
- league_tier (класс турнира)
