# dota_trader_v2 — Отчёт о собранных данных
**Дата:** 16 июня 2026  
**Контекст:** Система для поиска edge на рынке киберспортивных ставок. Стратегия — CLV (Closing Line Value): покупаем коэфициент дешевле чем Pinnacle закроет матч, зарабатываем на opening→closing line gap, а не на предсказании победителя.

---

## Заморожены (FREEZE_DATE = 2026-06-16)

- **Rule C (FROZEN):** `edge_adj > 0 AND elo_diff >= 75 AND odds < 2.0 AND market_prob 60–70%`
- **Elo модель:** PRIOR_WINS=22, PRIOR_LOSSES=4 — не трогать
- **H2H:** текущая формула — не трогать
- **Paper trading:** накапливаем 30 Rule C сигналов перед следующим исследованием. Сейчас 0/30.

## Скрипты (новые, этот сеанс)

### arb_scanner.py
Сканер арбитража через BetsAPI по 4 дисциплинам (Dota 2, CS2, LoL, Valorant).
- `--scan`: один проход, найти текущие арб-возможности
- `--watch --interval N`: мониторинг каждые N минут
- `--calc --o1 X --o2 Y --bank N`: расчёт ставок
- Хранит историю арб-возможностей в `data/arb_tracker.db`
- Порог: impled < 1.0 (то есть сумма обратных коэфов < 100%)

**Математика:**
```
implied = 1/odds_home + 1/odds_away
profit% = (1 - implied) * 100
stake_home = bank * (1/odds_home) / implied
stake_away = bank - stake_home
```

### matched_betting.py
Back-back matched betting (без exchange, два букмекера).
- 12 букмекеров предзагружены (GGBet, MelBet, FonBet, 1xBet, Bet365, etc.)
- `--calc --freebet F --o1 X --o2 Y`: расчёт
- `--add`, `--list`, `--use`, `--report`: управление бонусами
- БД: `data/matched_betting.db` (FUSE-safe: работает через /tmp)

**Математика (profit-only free bet):**
```
hedge = F * (odds_bm1 - 1) / odds_bm2
profit = F * (odds_bm1 - 1) * (odds_bm2 - 1) / odds_bm2
extraction% = profit / F * 100
```
При odds=2.0: извлечение 50% от суммы фрибета.  
При odds=3.0: извлечение 133%.
