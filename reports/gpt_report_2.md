# Dota 2 Quant Betting System — Отчёт #2 для GPT

## Контекст

Строим систему paper-trading для ставок на Dota 2. Цель — найти систематический edge против букмекерского рынка через обработку публичных данных лучше чем это делают остальные участники рынка.

**Стек:**
- Матчи и результаты: PandaScore API (~4000 Dota 2 матчей за 12 месяцев)
- Коэффициенты: BetsAPI (~16000 odds snapshots, 268 матчей с open+close)
- Модель: Walk-forward Elo (K=32, tier-weighted, time decay 365-day half-life)
- БД: SQLite

Проверили каждый признак по отдельности как предиктор результата матча:

| Признак | Accuracy | Brier | LogLoss | Corr с результатом |
|---|---|---|---|---|
| Form diff (last5/last10) | 0.623 | 0.2335 | 0.6596 | +0.255 |
| Elo probability | 0.642 | 0.2221 | 0.6357 | +0.328 |
| Market move (close−open) | 0.624 | 0.2158 | 0.6214 | +0.383 |

**Вывод:** Рынок открытия уже превосходит наш Elo (Acc 69.8% vs 64.2%, Brier 0.207 vs 0.222). Форма — самый слабый сигнал. Закрытие рынка — вне конкуренции (Acc 77%).

### 2. Комбинации признаков

| Комбинация | Brier | ΔBrier vs Elo |
|---|---|---|
| Elo-only (baseline) | 0.2221 | — |
| Elo 20% + Market open 80% | 0.2064 | **−7.1%** |
| Elo 30% + Form 10% + Market 60% | 0.2065 | −7.0% |
| Market open-only | 0.2067 | −6.9% |

Grid search показал оптимальный вес Elo = 20%, Market = 80%. Добавление Elo к рынку даёт только −0.3% Brier сверх Market-only — то есть Elo почти не добавляет информации поверх рынка.

### 3. H2H анализ

Добавили H2H win rate как фичу (с decay-weighting: матчи старше весят меньше):

| Модель | H2H cutoff | n | acc_model | acc_market |
|---|---|---|---|---|
| Elo-only | все | 228 | 0.632 | 0.706 |
| Elo 60% + H2H 40% | >= 2 встреч | 107 | 0.701 | 0.710 |
| Elo 60% + H2H 40% | >= 3 встреч | 86 | 0.709 | 0.721 |
| **Elo 60% + H2H 40%** | **>= 5 встреч** | **51** | **0.706** | **0.686** |

При H2H >= 5 встреч **модель впервые обогнала рынок по accuracy** (70.6% vs 68.6%). Корреляция edge с результатом стала положительной (+0.027 vs −0.080 у Elo-only в этом сегменте). n=51 — мало, нужно 200+.

## Текущее понимание проблемы

**Главная проблема — не отсутствие edge, а масштаб Elo.** Elo генерирует вероятности в диапазоне 0.35–0.75. Рынок ставит 0.80–0.96 на явных фаворитов. Когда Elo говорит 0.50, рынок уже знает что это 0.65–0.75 — и оказывается прав. Это проявляется как:
- Edge (model − market) систематически отрицательный для очевидных фаворитов
- corr(edge, result) = −0.31 потому что большие положительные edge сосредоточены там где рынок прав

**Что работает:**
- CLV+ = 54% в целом (рынок в 54% случаев движется в сторону Elo) — есть слабый сигнал
- CLV+ = 91.7% для сегмента elo_diff>200 + model≥market — сильный сигнал
- H2H >= 5 встреч: accuracy модели превышает accuracy рынка
- elo_diff > 200: ROI +32.7%, corr становится положительной

## Стратегия которую мы выбрали

**Одна точка где мы можем быть умнее рынка: H2H-скорректированный Elo.**

Рынок обрабатывает десятки матчей одновременно и на Tier A/B турнирах не всегда глубоко анализирует конкретную парную историю. Наша система делает это систематически для всех пар. Сейчас n=51 — собираем больше данных (BetsAPI по 21 Dota 2 лиге отдельно).

Формула H2H-blend:
```
если h2h_games >= 5:
    adjusted_prob = elo_prob * 0.60 + h2h_winrate_decay * 0.40
иначе:
    adjusted_prob = elo_prob
```

h2h_winrate_decay — взвешенный по времени WR: матч год назад весит 0.5^(days/365).

## Вопросы к GPT

**1. Feature importance и роль рынка в модели**

Рынок открытия сильно превосходит Elo как предиктор. Если мы добавим market_open_prob как фичу в нашу модель — это создаёт look-ahead bias? Мы считаем edge как (model − market), если model обучена на market — это circular reasoning. Как правильно включить рыночный сигнал не разрушив смысл edge?

**2. H2H и размер выборки**

При H2H >= 5 встреч на n=51 матчах модель превышает market accuracy (70.6% vs 68.6%). Какой минимальный n нужен чтобы считать этот сигнал статистически значимым? Насколько опасно что мы выбирали cutoff (5 встреч) post-hoc по данным?

**3. CLV как метрика качества**

CLV+ = 54% на всём датасете. Какой CLV+ считается достаточным для того чтобы система имела +EV на дистанции с учётом маржи букмекера ~4-5%? Есть ли публичные бенчмарки CLV для esports или других нишевых рынков?

**4. elo_diff > 200 сегмент**

ROI +32.7% на 29 матчах, corr(edge,result) = +0.14. Насколько это значимо? Bootstrap CI на n=29 скорее всего очень широкий. Как правильно оценить вероятность что это реальный сигнал, а не luck?

**5. Следующий приоритет при n~300 матчах**

Что из следующего важнее сделать сначала при увеличении датасета до 300+ матчей:
- (a) isotonic calibration Elo → откалиброванные вероятности
- (b) XGBoost второго уровня поверх Elo + H2H + form
- (c) продолжать накапливать данные без изменений в модели
- (d) что-то другое

## Технические детали модели

```python
# Walk-forward Elo (нет look-ahead bias)
K = 32  # базовый
# tier-weighted K: TI/Major → K*1.5, DPC/ESL → K*1.2, Qualifiers → K*0.7

# Time decay в обучении:
weight = 0.5 ** (days_ago / 365)

# Предсказание:
elo_prob = 1 / (1 + 10 ** ((elo_B - elo_A) / 400))

# H2H decay:
h2h_weight(match) = 0.5 ** ((now - match_date_days) / 365)
h2h_wr_decay = sum(weights * wins) / sum(weights)

# No-vig market prob:
p = (1/odd_home) / (1/odd_home + 1/odd_away)

# Edge:
edge_open = elo_prob - market_novig_open

# CLV:
CLV = market_close_prob - market_open_prob  # >0 = рынок движется в нашу сторону
```
