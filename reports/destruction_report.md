## Задача 1: Permutation Test ✓✓ УБЕДИТЕЛЬНО

**H0:** ROI Rule C можно получить случайным образом при случайных результатах матчей.

```
Null distribution (10 000 shuffle):
  Mean    = -0.1398  (не 0, т.к. bookmaker edge в odds<2.0 сегменте)
  P95     = +0.0866
  P99     = +0.1977
  P99.9   = +0.2707
  Observed= +0.2604

p-value (ROI):  0.0024  (0.24%)  ← p < 1%  ✓✓
p-value (WR):   0.0029  (0.29%)  ← p < 1%  ✓✓
```

WR под H0 (если рынок прав): 0.647. Реальный WR: 0.875. Превышение: **+0.228**.

**Вывод:** Rule C НЕ является случайным шумом. p=0.24% означает, что при 10 000 random shuffles только 24 из них дали ROI ≥ наблюдаемого.

**Важно:** Null mean = -0.14 (не 0). Это объясняется bookmaker edge в сегменте odds<2.0. Rule C превышает null p95 и стоит почти на границе null p99.9.

## Вопросы для GPT

**1. Null distribution mean = -0.14, не 0. Почему?**

В permutation test при shuffle результатов, Rule C продолжает выбирать те же ставки (те же odds<2.0). Bookmaker overround в odds<2.0 сегменте при 60-70% market_prob составляет ~5-10%. Поэтому под H0 ожидаемый ROI не 0, а отрицательный. Это правильная интерпретация? Или null должен быть ≈0?

Если null = -0.14, то Rule C beat null на +0.40 ROI (от -0.14 до +0.26). Это значительно больше чем разница от нуля (+0.26). Как правильно интерпретировать?

**2. Task 2: 56% случайных правил лучше Rule C по CI_lo. Критически ли это?**

Среди 265 валидных случайных правил 149 имеют CI_lo >= +0.042. Значит ли это:
- (a) Весь сегмент (odds<2.0, mkt 60-70%) прибыльный, и мы просто "плывём" в позитивном потоке
- (b) Топ-правила (с edge>=0.07) действительно лучше Rule C
- (c) Overfitting: у нас 534 матча и достаточно параметров чтобы найти прибыльные комбинации случайно

Как разграничить (a) от (c)? Особенно с учётом того что permutation test (Task 1) показал p<1%.

**3. Null distribution в Task 2: медиана ROI = +0.297. Если даже случайные правила дают +0.30, что это значит?**

Медиана случайных правил (n>=10) = +0.297, а Rule C = +0.260. Интерпретаций несколько:
- Весь сегмент odds<2.0 + mkt 60-70% прибыльный — market inefficiency в этой зоне универсальна
- Small sample bias: правила с n~10-20 при случайном отборе показывают inflated ROI
- Комбинация обоих факторов

Если медиана случайных правил выше Rule C, стоит ли вообще иметь правило, или просто ставить на весь сегмент odds<2.0 + mkt 60-70%?

**4. H2H causality: avg_elo_diff H2H positive (145) vs H2H neutral (155) — разница небольшая, но есть. Достаточно ли контроля?**

corr(h2h_delta, elo_diff) = -0.266. Слабый, но не нулевой. Для более строгого контроля нужен многовариантный анализ (multiple regression: ROI ~ h2h_delta + elo_diff). При n=24 это нереально. Есть ли другой способ доказать causality при малом n?

**5. Финальное решение: Rule C или Rule C+ (с edge>=0.07)?**

Данные указывают: добавление `edge_adj >= 0.07` улучшит результаты (Task 2, Task 4). Но это post-hoc тюнинг на тех же данных. Учитывая:
- n(Rule C) = 24, n(Rule C с edge>=0.07) ≈ 13-15
- При n=15 CI будет широким
- Permutation test показал значимость Rule C (p=0.24%)

Правильная ли стратегия: запустить paper daemon на Rule C (n будет расти), и только после 50+ новых ставок пересмотреть threshold для edge? Или безопаснее зафиксировать edge threshold сейчас?
