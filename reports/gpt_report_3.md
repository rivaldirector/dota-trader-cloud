## Методология — Walk-Forward с H2H

Все предсказания строго walk-forward: Elo обновляется после каждого матча, предсказание делается ДО обновления. H2H win rate тоже обновляется только после матча.

```python
# Smooth H2H blend:
h2h_conf   = min(h2h_n / 5.0, 1.0)          # 0.0 при 0 встречах, 1.0 при ≥5
h2h_weight = 0.4 * h2h_conf                   # макс вес H2H = 40%
adj_prob   = elo_prob * (1 - h2h_weight) + h2h_wr_decay * h2h_weight

# H2H decay:
w_wins  = Σ 0.5^((now_ts - match_ts) / 365 / 86400) for wins
w_total = Σ 0.5^((now_ts - match_ts) / 365 / 86400) for all
h2h_wr_decay = w_wins / w_total
```

## Текущее понимание проблемы

### Почему corr глобально отрицательный?

При elo_diff < 150 (437/538 матчей) Elo говорит ~0.55–0.65 для фаворита, а рынок ставит 0.75–0.90 — и рынок прав. Edge (elo − market) положительный, но направлен в сторону неправильной команды. Elo недооценивает явных фаворитов потому что у них ограниченная история матчей в нашем датасете (12 месяцев).

### Почему H2H помогает?

Рынок обрабатывает десятки матчей одновременно и не глубоко анализирует парную историю команд. Наша система делает это систематически с decay-weighting. Когда есть H2H история, модель точнее рынка на 7-9 процентных пунктов по accuracy.

### Где сигнал

Единственный доказанный сегмент: когда **Elo уверен (diff ≥ 150)** и **модель говорит что шансы выше рыночных** (edge_adj > 0). В этом сегменте всё сходится: accuracy, CLV, corr, ROI.

## Стратегия ставок (paper trading)

**Ставить:** только когда edge_adj > 0 AND elo_diff ≥ 150
**Не ставить:** при elo_diff < 150 (corr отрицательный, ROI значимо отрицательный)

Покрытие: 50/538 = 9.3% всех матчей. При 538 матчах за 12 месяцев → ~4 ставки в месяц. Нужно ещё 6–12 месяцев чтобы накопить проспективный n=50+.

## Вопросы к GPT

### 1. Multiple testing / data dredging

Мы перебирали 14 комбинаций сегментов. Единственная статистически значимая — [+edge_adj & elo≥150]. Насколько серьёзна проблема множественного тестирования при bootstrap CI? Нужна ли поправка Бонферрони? После поправки на 14 тестов порог станет 0.05/14 = 0.0036 — это уничтожит значимость?

### 2. CLV как независимое подтверждение

CLV+ = 0.640 в нашем сегменте. Это означает что рынок в 64% случаев после нашей точки наблюдения двигается в нашу сторону (closing line движется к нашей вероятности). Является ли CLV+ = 0.640 сильным независимым подтверждением что edge реален? Какой CLV+ считается значимым в литературе по betting?

### 3. n=50 и проспективная валидация

Для проспективной валидации сигнала нужен независимый тест. При n=50 и coverage 9.3% нам нужно примерно 540 матчей исторических данных чтобы набрать новые 50 тест-матчей. Есть ли способ быстрее? Можно ли разбить существующий датасет на train/test по времени чтобы получить честную оценку?

### 4. ROI +0.440 — не слишком ли высокий?

ROI = +0.440 на n=50 матчах выглядит подозрительно высоким для ставок. Типичный edge в спортивном беттинге — 3-8% ROI на дистанции. Либо это действительно post-hoc overfitting, либо в нашем сегменте есть структурная неэффективность рынка. Как отличить одно от другого при n=50?

### 5. Следующий приоритет

Что делать дальше:
- (a) Накапливать больше данных (ждать 3-6 месяцев) и не трогать модель
- (b) Разбить датасет 70/30 по времени, re-validate на holdout
- (c) Добавить новые признаки (roster changes, patch impact, recent form)
- (d) Попробовать calibration для сегмента elo_diff ≥ 150 отдельно

## Технические детали

```python
# Walk-forward Elo (строго без look-ahead)
BASE_K = 32
K_multipliers: S=1.5, A=1.2, B=1.0, C=0.7
Time decay: k_eff = k * 0.5^(days_ago/365)

# Prediction (до обновления Elo):
elo_prob = 1 / (1 + 10^((elo_B - elo_A) / 400))

# H2H-adjusted:
h2h_conf   = min(h2h_n / 5.0, 1.0)
adj_prob   = elo_prob*(1-0.4*h2h_conf) + h2h_wr_decay*0.4*h2h_conf

# Edge:
edge_adj = adj_prob - market_novig_open

# No-vig:
p = (1/odd_home) / (1/odd_home + 1/odd_away)

# ROI метрика:
roi = mean((result*2-1) * sign(edge))  # +1 если угадали, -1 если нет

# Bootstrap 95% CI:
5000 resamples с replacement, percentile [2.5, 97.5]
```
